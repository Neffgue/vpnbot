# 🚨 АГРЕССИВНОЕ ТЕХНИЧЕСКОЕ ЗАДАНИЕ (MASTER DIRECTIVE) 🚨

**ЦЕЛЕВАЯ МОДЕЛЬ:** Claude 4.6 Sonnet
**ПРОЕКТ:** Коммерческий VPN-сервис (Telegram Bot + React Admin Panel + FastAPI/Python Backend) на базе Marzban (Xray) для клиента HAPP VPN.
**СТАТУС:** КРИТИЧЕСКИЙ. Текущая архитектура синхронизации сломана. Присутствуют фатальные баги (500, Connection Reset). UI/UX не соответствует требованиям.

**ТВОЯ ЗАДАЧА КАК АГЕНТА:** Полностью устранить рассинхрон между веб-админкой и ботом. Бот должен стать абсолютно "глупым" клиентом, который ВСЕ данные, тексты, цены и картинки берет ИСКЛЮЧИТЕЛЬНО из Базы Данных, которой управляет веб-админка. Никакого хардкода. 

Ниже приведены жесткие требования по 5 фазам. Выполняй их последовательно.

---

## ФАЗА 1: Устранение критических сбоев API (Бэкенд)

Сейчас панель управления мертва. При изменении цен (например, `/admin/plan-prices`) или загрузке страниц падают ошибки:
* `net::ERR_CONNECTION_RESET` на `GET /api/v1/admin/plans`
* `500 Internal Server Error` на `PUT /api/v1/admin/plans/...`

**ДИРЕКТИВЫ:**
1. **Жесткий Exception Handling:** Внедрить глобальный обработчик исключений в FastAPI. Бэкенд НИКОГДА не должен падать с 500 ошибкой в тишину или сбрасывать соединение. Любая ошибка БД (SQLAlchemy) должна ловиться, делать `db.rollback()` и возвращать фронтенду JSON (код 400/422) с текстом ошибки.
2. **Починка CRUD Тарифов:** Полностью переписать логику обновления тарифов (`PUT`). Убедиться, что типы данных (UUID, Integer, Decimal) совпадают между React-фронтендом, Pydantic-схемами и SQLAlchemy моделями. Цены ДОЛЖНЫ сохраняться в БД.
3. **Единый источник истины (SSOT):** Бот больше не имеет права хранить цены в своем коде. Когда пользователь нажимает "Оплатить", бот делает запрос в БД (или дергает внутренний сервис), берет АКТУАЛЬНЫЕ цены и формирует инлайн-кнопки на лету.

## ФАЗА 2: Медиа-Движок и UI Бота (Текстовое описание дизайна)

Ты не видишь картинки, поэтому читай внимательно. Бот должен выглядеть премиально.

**ДИРЕКТИВА 1: Изображения для КАЖДОГО меню**
Бот должен поддерживать отправку сообщений в формате: `[ИЗОБРАЖЕНИЕ] -> [ПОДПИСЬ К ИЗОБРАЖЕНИЮ] -> [КНОПКИ]`.
* **Что сделать в БД:** В таблицу текстов/меню добавить поле `media_url` или `media_path`. 
* **Что сделать в Админке:** Для каждого раздела (Приветствие /start, Личный кабинет, Поддержка, Тарифы) добавить поле `File Upload` (Загрузить картинку). 
* **Починить 404:** Изображения сохраняются в `static/uploads/`. Настрой Docker Volume, чтобы Nginx корректно отдавал файлы по URL, иначе бот их не скачает!
* **Логика бота:** Бот должен использовать `message.answer_photo(photo=..., caption=..., reply_markup=...)`. Если фото в БД нет — отправлять обычный текст.

**ДИРЕКТИВА 2: Форматирование "Личного Кабинета" (QUOTE style)**
Профиль пользователя должен выглядеть сочно, используя HTML-теги `<blockquote>` или MarkdownV2. 
**Строгий шаблон (реализовать 1 в 1):**
```markdown
> 👤 **Профиль:**
> > Имя: {user.name}
> > ID: {user.id}
> > 💳 Баланс: {user.balance} ₽

**🔑 Ваша подписка:**
`{sub_link}`

> **📦 Информация о тарифе:**
> > 📋 Группа: 👥 {plan.name}
> > 💎 Тариф: {plan.duration}
> > 📱 Лимит устройств: {plan.device_limit}
> > 🔄 Привязанных устройств: {devices_connected}

📆 Срок действия: {expire_date}
```

**ДИРЕКТИВА 3: Одноразовая кнопка "Пробный период"**
* Логика: В таблице `users` добавить флаг `has_used_trial = Boolean(default=False)`.
* При нажатии на "Пробный период", выдавать триал, ставить `True` в БД и обновлять меню.
* **ВАЖНО:** Не отправлять сообщение "Вы уже использовали триал". Кнопка должна просто ИСЧЕЗНУТЬ из клавиатуры (`reply_markup`) навсегда для этого пользователя.

## ФАЗА 3: Интеграция с Marzban и приложением HAPP

Проект работает на ядре Xray через панель Marzban. Клиенты используют приложение HAPP.

**ДИРЕКТИВА 1: Управление серверами (Нодами) через Админку**
Необходимо создать новую вкладку в React-админке "Серверы (VPN Nodes)".
* **Поля БД:** `ID`, `Имя Сервера`, `IP/Домен`, `Marzban API URL`, `Marzban Username`, `Marzban Password/Token`, `Is_Active`.
* **Суть:** Бэкенд должен уметь подключаться к разным панелям Marzban по API, балансировать юзеров или позволять админу выбирать, на каком сервере создать подписку.

**ДИРЕКТИВА 2: Лимиты устройств (Device Limits)**
Marzban API поддерживает ограничение по количеству устройств (Sub Limit / IP Limit).
* Тариф "Соло" -> отправляем API запрос в Marzban с `limit=1`.
* Тариф "Семья" -> отправляем API запрос в Marzban с `limit=5`.
Если пользователь превышает лимит, Marzban сам заблокирует его.

**ДИРЕКТИВА 3: Вкладка "Мои устройства" (Детализация)**
В личном кабинете при нажатии "Мои устройства" бот должен запрашивать данные подписки и выводить информацию о подключениях:
```text
💻 Ваше устройство:
🔑 HWID: {device.hwid}
└ 📱 Модель: {device.os}
└ 🌐 User-Agent: Happ/3.11.0
└ 🕒 Подключено: {connect_time}
[Кнопка: Удалить устройство]
```
*(Для реализации этого используй статистику подключений из API Marzban или логику привязки Sub-link к HWID, если Happ передает такие заголовки при запросе конфига).*

## ФАЗА 5: Локализация и Финальная синхронизация

1. **Русский язык:** Проверить весь бот от и до. Никаких английских системных сообщений, ошибок или дат в формате `YYYY-MM-DD`. Все перевести на русский (например: `18 марта 2026 года`).
2. **Абсолютная синхронизация:** Я (владелец) должен иметь возможность зайти в веб-админку, изменить текст "Личного кабинета", прикрепить туда картинку миньона, изменить цену тарифа "Соло" на 199 руб — и при следующем нажатии кнопок в боте, пользователь ДОЛЖЕН увидеть эти изменения МГНОВЕННО, без перезагрузки бота. Реализуй это через прямые запросы в БД при навигации пользователя по боту.
