#### Конкретные задачи

1. **Исправить отображение профиля**  
   - Почему у пользователя с балансом 1150 ₽ нет активной подписки? Возможно, данные о подписке не подгружаются или неверно отображаются.  
   - Сравнить структуру ответа API для профиля с ожидаемой (изображение 1) и текущей (изображение 2).  
   - Проверить, приходит ли с сервера поле с подпиской. Если нет — найти причину в бэкенде.  

2. **Исправить кодировку в сообщении с тарифами**  
   - Проблема: при нажатии «Оплатить тариф» бот отправляет текст, который отображается кракозябрами.  
   - Наиболее вероятная причина: сервер отдаёт текст в кодировке Windows-1251, а клиент (Telegram) ожидает UTF-8, либо наоборот.  
   - Необходимо проверить, в какой кодировке текст хранится в базе/конфигах, и убедиться, что при отправке через Telegram API используется корректная UTF-8 строка.  
   - Возможно, проблема в HTTP-заголовках или в настройках самого бота (например, при чтении из файла или БД без преобразования).  

3. **Добавить возможность назначать изображения на кнопки (и в целом на любые пункты меню) через веб-админку**  
   - Требуется, чтобы администратор мог в панели управления выбрать для каждой кнопки (например, «Оплатить тариф», «Личный кабинет», «Инструкция», и т.д.) изображение, которое будет отправляться вместе с текстом.  
   - Пример поведения: пользователь нажимает кнопку → бот присылает фото, под ним текст, а затем снова кнопки (или та же клавиатура).  
   - Реализовать хранение соответствий: `button_id` → `image_url` (или file_id из Telegram).  
   - В админке должен быть удобный интерфейс загрузки/выбора изображения для каждого элемента меню.  
   - Учесть, что в Telegram изображение можно отправить либо как отдельное медиа, либо в составе группы. Лучше использовать `InputMediaPhoto` с подписью.  
   - При изменении изображения в админке бот должен автоматически использовать новое при следующем вызове кнопки.  

4. **Починить ошибки API при редактировании цен тарифов**  
   - Ошибка 500 (Internal Server Error) на POST `/api/v1/admin/plans` — вероятно, проблема на стороне сервера: неверный формат запроса, отсутствие обязательных полей, или ошибка в логике обработки.  
   - Ошибка `ERR_CONNECTION_RESET` на PUT — возможно, сервер падает или закрывает соединение из-за слишком большого payload, таймаута или CORS-политики.  
   - Необходимо:  
     a. Посмотреть точный запрос, который уходит в момент сохранения (из вкладки Network в DevTools).  
     b. Сравнить с ожидаемой схемой API (вероятно, нужно передавать массив тарифов или объект с ценами).  
     c. Проверить логи сервера на Alwaysdata (если доступны).  
     d. Убедиться, что админка авторизована (возможно, требуется токен или сессия, а запрос идёт без них — тогда сервер ресетит соединение).  
     e. Исправить либо клиентскую часть (чтобы отправляла корректные данные), либо серверную (чтобы правильно обрабатывала).  

5. **Общие рекомендации**  
   - Добавить обработку ошибок на клиенте и сервере с понятными сообщениями для администратора.  
   - Привести все тексты в боте к единой кодировке UTF-8.  
   - Рассмотреть возможность использования кеширования изображений (file_id) для ускорения работы бота.  
   - Протестировать изменения на тестовом боте перед выкаткой на продакшн.
