#### **Задача 1: Реализация управления URL-ссылками в инструкциях**

*   **Описание проблемы:** В разделе "Инструкция" в боте существуют пошаговые руководства. В некоторых шагах есть кнопки, предназначенные для скачивания приложений, например, кнопка с текстом "Скачать из Google Play" и иконкой ↗, и кнопка "Скачать APK". На данный момент ссылки, стоящие за этими кнопками, невозможно изменить из админ-панели.

*   **Требуемая реализация:**
    1.  В веб-админ панели, в интерфейсе редактирования инструкций, необходимо добавить функционал "Кнопки-ссылки".
    2.  Для каждого шага инструкции администратор должен иметь возможность добавить одну или несколько кнопок.
    3.  Для каждой такой кнопки должно быть два поля для редактирования: "Текст кнопки" (например, "Скачать из Google Play") и "URL-ссылка" (например, `https://play.google.com/store/apps/details?id=...`).
    4.  Изменения, сохраненные в веб-панели, должны немедленно применяться в боте. Если администратор изменит URL у кнопки, пользователи в боте при нажатии на нее уже будут переходить по новой ссылке.

#### **Задача 2: Исправление ошибки "500 Internal Server Error" при изменении цен на тарифы**

*   **Описание проблемы:** Попытка изменить цену тарифа через `/admin` интерфейс в боте или через веб-панель завершается ошибкой. Администратор видит сообщение: `Ошибка обновления цены: Server error '500 Internal Server Error' for url 'https://neffgue313.alwaysdata.net/api/v1/admin/subscriptions/plans/Solo'`. Это означает, что сервер не может обработать запрос на обновление. При этом в боте для пользователей отображается статичный список цен (например, "7 дней — 150 рублей", "1 месяц — 250 рублей" и т.д.), который не меняется.

*   **Требуемая реализация:**
    1.  **Backend:** Провести отладку серверного эндпоинта `PUT /api/v1/admin/subscriptions/plans/{id}`. Необходимо найти и устранить причину внутренней ошибки сервера (500). Вероятные причины: неверный SQL-запрос, проблема с правами доступа к базе данных, ошибка валидации на стороне сервера.
    2.  **Frontend/Backend:** Обеспечить, чтобы после исправления ошибки, цена, измененная в веб-панели для конкретного периода (например, цена для "7 дней" в тарифе "Семейный"), корректно сохранялась в базе данных.
    3.  **Синхронизация:** Убедиться, что бот при отображении списка тарифов для оплаты всегда запрашивает актуальные цены из базы данных, а не использует зашитые в код значения.

#### **Задача 3: Реализация функции "Изображение-обложка" для каждого пункта главного меню**

*   **Описание проблемы:** Сейчас при нажатии на кнопки главного меню ("Личный кабинет", "Оплатить тариф" и т.д.) бот отправляет только текстовое сообщение и клавиатуру с новыми кнопками. Это выглядит менее привлекательно, чем в эталонных примерах.

*   **Требуемая реализация:**
    1.  **Backend/API:** Создать механизм, позволяющий привязать одно изображение к каждому сообщению, которое отправляется при нажатии на кнопку главного меню.
    2.  **Web-Admin:** В веб-админ панели, в разделе редактирования сообщений бота (например, "Тексты бота" или "Кнопки меню"), для каждого элемента ("Приветствие /start", "Текст личного кабинета" и т.д.) добавить поле "Изображение-обложка" с функцией загрузки файла.
    3.  **Bot Logic:** Изменить логику отправки сообщений. Теперь, когда пользователь нажимает, например, "Личный кабинет", бот должен отправить **единое сообщение**, которое содержит:
        *   **Сверху:** Загруженное в админ-панели изображение-обложку.
        *   **Под изображением:** Текстовое описание ("Ваша личная информация..." и т.д.).
        *   **В самом низу:** Клавиатуру с соответствующими кнопками ("Мои платежи", "Мои устройства" и т.д.).
    4.  **Важно:** Убедиться, что загрузка изображений работает и не вызывает ошибок `404 Not Found` (см. лог ошибок из предыдущих заданий). Это требует правильной настройки путей для сохранения и отдачи статических файлов.

*   **Описание проблемы:** Раздел "Инструкции" в админ панели не работает. Он либо сообщает "инструкции не найдены", либо не позволяет загружать изображения к шагам.

*   **Требуемая реализация:**
    1.  **Синхронизация:** В первую очередь, исправить API-запрос, чтобы панель корректно находила и отображала для редактирования уже существующие в боте тексты инструкций.
    2.  **Интерфейс в Admin:** Реализовать пошаговую структуру, как в эталонном примере. Интерфейс должен позволять:

#### **Задача 5: Исправление бага: исчезновение кнопок меню после загрузки изображения через /admin**

*   **Описание проблемы:** Был обнаружен критический баг. После того как администратор через команду `/admin` в боте загрузил изображение для приветственного сообщения (`/start`), из клавиатуры главного меню пропала кнопка "Личный кабинет".

*   **Последовательность, приведшая к ошибке:**
    1.  Администратор использовал команду `/admin`, перешел в управление изображениями для кнопок.
    2.  Было успешно загружено изображение (в примере — миньон).
    3.  После этого действия, при вызове команды `/start`, бот стал показывать новое изображение, но в составе кнопок под ним отсутствовала кнопка "Личный кабинет".

*   **Требуемая реализация:**
    1.  Провести ревизию кода, отвечающего за обновление и построение клавиатуры главного меню.
    2.  Логика, которая привязывает изображение к сообщению, **не должна** затрагивать или изменять массив/структуру кнопок, привязанных к этому же сообщению.
    3.  Необходимо убедиться, что функция обновления/загрузки изображения является **атомарной** и модифицирует только поле "photo" или аналогичное, не затрагивая поле "reply_markup" (разметка клавиатуры).
